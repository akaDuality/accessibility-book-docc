# Вертикальные свайпы

Горизонтальные свайпы переключают фокус между элементами, а вертикальные — меняют значение внутри элемента.

## Overview

В предыдущих главах мы подписали элементы и уменьшили их количество, так мы сделали использование приложения возможным и даже удобным. По экрану теперь можно перемещаться горизонтальными свайпами и все элементы будут читаться.

В этой главе обсудим вертикальные свайпы. С их помощью можно выполнять действия над элементами в фокусе, например:
- поменять значение внутри элемента (как в UISlider или UIStepper),
-  сменить действие с активации на другое (как действия в ячейках, которые скрываются
за свайпом),
- рассказать подробнее о текущем элементе.

Разберем все случаи подробно, ведь они могут сильно повысить удобство
использования приложения.

## Трейт .adjustable

![Степпер, слайдер и сегмент контрол управляются вертикальными свайпами](verticalSwipes-samples)

До этого момента мы использовали только один трейт поведения кнопки — `.button`. Он добавлял к описанию текст «кнопка» и по двойному тапу вызывал метод `accessibilityActivate()`.

Какие элементы сложнее чем кнопка? Например, `UIStepper`, `UISegmentedControl` и `UISlider`. На первый взгляд элементы очень разные, но работу всех трех элементов можно свести до одного правила: контрол позволяет выбрать значение. Разница только в том, что они дают разный контроль над точностью изменений:
- `UIStepper` изменяет значение на ближайшее;
- `UISlider` позволяет выбрать в широком диапазоне, но с небольшой точностью;
- `UISegmentedControl` показывает все значения, можно переключиться к любому.

Если мы еще сильнее упростим модель, то значение внутри контрола можно уменьшить или увеличить. Это довольно частый паттерн, поэтому у VoiceOver есть для этого специальный модификатор поведения — трейт .adjustable.

Для переключения между элементами мы используем горизонтальные свайпы: к
следующему и предыдущему. При переключении читается лейбл.

А что, если вертикальные свайпы будут влиять на внутреннее состояние контрола? Например, мы бы могли с их помощью менять значение, записанное в `.accessibilityValue`? Тогда свайп «вверх» мог бы увеличивать количество, а «свайп» вниз уменьшать его.

За это отвечает трейт `.adjustable`. Когда вы добавляете его к элементу, происходит три вещи:

1. К описанию элемента добавляется фраза **«элемент регулировки»**.
2. VoiceOver подсказывает жест: «Смахните вверх или вниз одним пальцем, чтобы настроить значение».
3. При вертикальном свайпе вызываются методы `accessibilityIncrement()` или `accessibilityDecrement()`.

После свайпа VoiceOver не перечитывает всё описание элемента — **читается только новое значение**. Это делает переключение между значениями быстрым и ненавязчивым.

## UIStepper

![Примеры степперов](verticalSwipes-stepper)

Степпер может быть разных форм, но суть остается одна: у нас есть какое-то значение и мы влияем на него с небольшим шагом. Иногда надпись рядом, иногда внутри. В любом случае, сделаем одним элементом: назовем, отделим изменяемое значение и расскажем, как взаимодействовать с ним.

- **Название.** Мы меняем количество элементов, можем так и написать: «количество».
- **Значение.** Хватит и цифры, но можно превратить в полноценный текст: «3 события».
- **Трейт.** Можно задать трейт .adjustable, тогда сможем регулировать количество вертикальными свайпами, а после наших изменения VoiceOver прочитает новый value.

```
label: Количество,
value: 3 события,
trait: элемент регулировки.
```

Превратим это описание в код. Нужно сделать весь степпер доступным элементом, дать ему название и указать тип `.adjustable`.

```swift
class CustomSlider: UIView {
    var count: Int = 0
    
    override func awakeFromNib() {
        super.awakeFromNib()
        
        isAccessibilityElement = true
        accessibilityLabel = "Количество"
        accessibilityTraits = .adjustable
    }
    
    // Значение можно сделать вычисляемым полем:
    override var accessibilityValue: String? {
        get { return "\(count) события" }
        set {}
    }
    
    // Эти функции вызываются вертикальными свайпами
    // Свайп вверх увеличивает количество
    override public func accessibilityIncrement() {
        count += 1
        count.limit(by: 0...10)
    }
    
    // Свайп вниз уменьшает
    override public func accessibilityDecrement() {
        count -= 1
        count.limit(by: 0...10)
    }
}
```

Теперь VoiceOver читает: «Количество, 3 события, элемент регулировки». Свайп вверх увеличивает значение, свайп вниз — уменьшает. После каждого свайпа читается только новое значение: «4 события», «5 событий».

Если значение не меняется после свайпа — например, достигнут конец диапазона — VoiceOver проигрывает характерный звуковой сигнал. Пользователь сразу понимает, что дальше крутить нельзя.

> Note: Странно, что UIStepper по умолчанию не является элементом регулировки, он очень подходит для этого сценария.

### Степпер с текстом

![Текст около степпера нужно перенести в название степпера](verticalSwipes-stepperWithText)

Часто рядом со степпером есть поясняющий текст. Визуально такой контрол мы воспринимаем как единой целое, поэтому и для скринридера нужная такая же адаптация. UILabel надо скрыть от VoiceOver, а фрейм фокуса увеличить, чтобы он оборачивал и подпись и степпер.

Вместе с количеством товара часто меняется и общая стоимость. Расскажите о новой цене вместе с количеством через .accessibilityValue.

```swift
override var accessibilityValue: String? {
    get {
        let totalPrice = count * price
        return "\(count), \(totalPrice) рублей"
    } set {}
}    
```

> Tip: объединяйте контролы по смыслу, даже если визуально это несколько элементов

## UISlider

![Пример слайдера скорости речи](verticalSwipes-slider)

Слайдер работает по тому же принципу, что и степпер: вертикальный свайп меняет значение. Каждый свайп изменяет значение примерно на **10%** от общего диапазона.

Хорошая новость — `UISlider` уже имеет трейт `.adjustable` по умолчанию. Вам нужно только задать ему `accessibilityLabel`:

```swift
slider.accessibilityLabel = "Скорость речи"
```

VoiceOver прочитает: «Скорость речи, 50%, элемент регулировки».

Если шаг в 10% слишком грубый, есть альтернативный жест: **двойной тап с удержанием**. После двойного тапа не отпускайте палец — слайдер перейдёт в режим тонкой настройки. Двигайте палец влево или вправо, чтобы плавно менять значение, как мы делаем это обычно. 

> Tip: Если рядом уже есть текст с названием, скройте его от VoiceOver, чтобы он два раза не читал одно и то же вслух.

## UISegmentedControl

![Сегмент-контрол выбора типа теста: традиционное или тонкое](verticalSwipes-segmented-simple)

VoiceOver считает, что `UISegmentedControl` — это группа кнопок, поэтому сообщает об их количестве в `accessibilityValue`:
```
label Традиционное,
value 1 из 2,
trait Кнопка.
```

Но VoiceOver ничего не знает о контексте, поэтому для всей группы стоит уточнить название через `label`, а текущее значение перенести в `value`:
```
label Тесто,
value Традиционное, 1 из 2,
trait Кнопка.
```

Теперь при фокусе VoiceOver озвучит: «Тесто, тонкое, 1 из 2. Кнопка». Стало понятнее.

### Сегмент-контрол с вертикальными свайпами

![Сегмент-контрол выбора размера пиццы: маленькая, средняя, большая](verticalSwipes-segmented-size)

UISegmentedControl можно превратить в элемент регулировки. Вместо нескольких кнопок пользователь получает один элемент и переключает значения вертикальными свайпами:
```
label Размер,
value Средняя, 2 из 3
trait Элемент регулировки.
```

Реализация в коде:

```swift
let sizes = ["Маленькая", "Средняя", "Большая"]
private var selectedIndex = 1 

override func awakeFromNib() {
    super.awakeFromNib()
    
    isAccessibilityElement = true
    accessibilityLabel = "Размер"
    accessibilityTraits = .adjustable
}

override var accessibilityValue: String? {
    get {
        return sizes[selectedIndex]
    } set {}
}  
    
// Эти функции вызываются вертикальными свайпами
// Свайп вверх увеличивает количество
override public func accessibilityIncrement() {
    selectedIndex += 1
    selectedIndex.limit(by: 0...2)
}

// Свайп вниз уменьшает
override public func accessibilityDecrement() {
    selectedIndex -= 1
    selectedIndex.limit(by: 0...2)
}
```

}

> Tip: Получается, что для визуально одинаковых элементов может быть разное поведение VoiceOver, потому что у них разная ментальная модель. Вау!

## Контекстные действия

![Ячейка в корзине и в меню](verticalSwipes-contextActions)

Адаптируем ячейку из корзины. Внешне она очень похожа на ячейку меню: тоже есть название, цена и описание. Для ячейки меню мы объединили название с ценой, а описание перенесли в value. C корзиной можно было бы поступить так же, но у нее есть значимое отличие — контрол с количеством.

Изменяя количество, мы меняем и цену. После изменения значения VoiceOver прочитает именно .value, количество и цена должны оказаться там. Все остальное не изменяется и поэтому будет в label. Описание такое:

```
label Цезарь, средняя 30 сантиметров, традиционное тесто,
value 1 штука, 695 рублей.
```

До этого мы всегда укрупняли элементы в ячейки до одного контрола, но нам важно не потерять переключатель размера, им ведь нужно как-то управлять. Есть еще один нюанс: для зрячих пользователей доступно действие «удалить», которое скрывается за свайпом. Сделать такой свайп из VoiceOver неудобно, действие оказывается недоступным.

![Действие удалить скрывается за горизонтальным свайпом](verticalSwipes-cartCell)

У ячеек такие действия за свайпом вполне стандартны для iOS, поэтому для них есть специальная адаптация: когда фокус попадает на такую ячейку, VoiceOver добавляет в конце описания:

```
Доступны действия: активировать (выбрано), удалить.
Чтобы переключить действия, смахните вверх.
```

> Tip: вертикальные свайпы подходят не только для управления элементом регулировки, но и могут переключать действие с «активации» (`accessibilityActivate`) на что-то другое. Выбрать другое действие можно вертикальным свайпом, а активировать двойным тапом.

Вернемся к выбору количества. Это два действия: уменьшить и увеличить, но выбрать можно одно из четырех:
- нажать на ячейку (может ничего не происходить, а можно, например, отредактировать товар),
- прибавить,
- убавить,
- удалить.

Полный текст ячейки будет такой:
```
label Цезарь, средняя 30 сантиметров, традиционное тесто,
value 1 штука, 695 рублей.
Доступны действия: активировать (выбрано), добавить, убавить, удалить.
```

Теперь надо разобраться, как эти действия добавлять. Код прямолинеен: создаем действие, даем ему имя и функцию, что обработает ее, а затем добавляем этот объект в customActions:

```swift
let increase = UIAccessibilityCustomAction(
    name: "Добавить",
    actionHandler: { [unowned self] (action) -> Bool in
        // ... Увеличить количество
        return true
    })

let decrease = UIAccessibilityCustomAction(
    name: "Убавить",
    actionHandler: { [unowned self] (action) -> Bool in
        // ... Уменьшить количество
        return true
    })

let delete = UIAccessibilityCustomAction(
    name: "Удалить",
    actionHandler: { [unowned self] (action) -> Bool in
        // ... Удалить из корзины
        return true
    })

accessibilityCustomActions = [increase, decrease, delete]
```

![Пример поста из социальной сети](verticalSwipe-socialPost)

Этот же подход отлично работает для **постов в социальных сетях**. У поста может быть множество кнопок: лайк, комментарий, репост, сохранить, меню. Все эти кнопки становятся контекстными действиями одной ячейки. Пользователь не блуждает по десяткам кнопок, а быстро переключается между действиями вертикальным свайпом.

```
label Dodo Engineering,
value Доступность на iOS началась с «36 секунд»…
Доступны действия:
— открыть ссылку (выбрано),
— лайкнуть,
— прокомментировать,
— поделиться,
— перейти к «Dodo Engineering»,
— информация (по двойному тапу прочитать количество просмотров и время записи).
```

> Note: Действия могут быть очень сложными и даже вложенными. Пример можно посмотреть [в ролике Apple про создание папок на рабочем столе](https://www.youtube.com/watch?v=w2Ds-I2L6PI).

## Большие описания

![Пример ячейчики заказа с временем, адресом, продуктами и кнопкой повторить](verticalSwipes-customContent)

Не все ячейки можно упростить до одного элемента доступности, иногда контролов внутри ячейки слишком много. Например, в истории заказов мы выводим номер, дату, тип заказа, адрес, список из 5-6 продуктов, сумму и еще пару действий: повторить заказ и открыть детальное описание заказа. 

В таком случае есть два варианта:
1. Немного укрупнить элементы, объединив в пары: номер и дату, тип и адрес, текст «сумма» и число. Проходить по ним придется по-отдельности, но это терпимо.

2. Использовать AXCustomContent, тогда при фокусе на ячейку VoiceOver будет читать только одно из значений, а остальные можно прослушать, свайпая вертикально. Такое описание можно посмотреть у фотографий в стандартном приложении «Фото»: iOS выводит дату, время, ориентацию фото, кто или что изображено и т.д.

Для таких случаев есть `AXCustomContent`. Основная информация читается сразу при фокусе, а детали доступны через вертикальные свайпы.

```swift
    AXCustomContent(label: "Сумма",
                    value: "1239 рублей")
```

Может быть неудобно первый раз, но сильно ускоряет навигацию в последующие разы.
```
1239 рублей, Сумма
```

Чтобы описание заработало, нужно реализовать протокол `AXCustomContentProvider` и завести свойство `accessibilityCustomContent`. Интересно, что протокол находится во фреймворке `Accessibility`, хотя до этого весь код доступности был в `UIKit`. Не забудьте импортировать фреймворк.

```swift
import Accessibility
@available(iOS 14.0, *)
extension OrderHistoryItemCell: AXCustomContentProvider {
    var accessibilityCustomContent: [AXCustomContent] {
        get {
            [AXCustomContent(label: "Сумма",
                             value: "1239 рублей"),
             AXCustomContent(label: "213",
                             value: "Номер заказа"),
             AXCustomContent(label: "Дата",
                             value: "24 апреля 2021 г. в 17:25"),
             AXCustomContent(label: "На доставку",
                             value: "Москва, Главный офис Додо, Ленинская Слобода, 19с7"),
             productsContent(products: ["Пицца Додо Микс", "Яблочный пирог"])]
        }
        set {}
    }
...
```

CustomContent опирается не на порядок элементов, а на их важность. Продукты
кажутся самым важным, надо читать их в первую очередь, поэтому я задал им высокий
приоритет. Название продуктов я склеил в одну строчку через запятую, этого достаточно.

```swift
...
    private func productsContent(_ products: [String]) -> AXCustomContent {
        let content = AXCustomContent(
           label: "Товары",
           value: products.joined(separator: ", "))
        
        content.importance = .high

        return content
    }
}    
```

### Подытоживающие кнопки

![Кнопка «В корзину» может подытожить настройки экрана](verticalSwipes-summary)

Такое описание хорошо подойдет и для подытоживающих кнопок. Например, вы настроили пиццу: выбрали размер, убрали лук, добавили побольше мяса. Перед добавлением в корзину хочется все проверить, но не проходить снова по всему экрану. Описание на кнопке добавления сильно упростит работу.

```
label В корзину за 754 рубля,
value Пицца Песто, средняя, тонкое тесто,
traits Кнопка.
customContent:
- Лук, убрали,
- Острый цыпленок, Добавили.
```

### Группировка описания
![Пример множественного описания для пищевой ценности](verticalSwipes-Energy)

Еще один пример использования, когда данных очень много. На скриншоте энергетическая ценность для комбо-набора из нескольких продуктов. Горизонтальные свайпы будут переключать продукт, а вертикальные свайпы читать одно из описаний. С помощью свойства .importance можно управлять порядком продуктов, например, сначала читать вес.

CustomContent может работать вместе с customActions и даже в .adjustable
элементах. В каком режиме будут работать вертикальные свайпы можно выбрать через ротор.

## Разные действия на одном экране

![Экран корзины](verticalSwipes-cart)

На одном экране могут сочетаться разные подходы. Возьмём экран корзины: для количества товара используем `customAction` (добавить, убавить, удалить), а для количества соуса — трейт `.adjustable` (просто больше или меньше).

Такой подход значительно сокращает количество элементов. Экран корзины, на котором было **31 элемент** для VoiceOver, после адаптации превращается в **6 элементов**. Навигация становится быстрой и понятной.

Выбирайте инструмент по задаче:

- **`.adjustable`** — когда нужно менять одно значение по шагам (количество, громкость, размер).
- **`customAction`** — когда у элемента несколько разных действий (добавить, удалить, поделиться).
- **`AXCustomContent`** — когда в элементе много информации и её нужно читать по частям.

```
    Додо Микс: Средняя 30 сантиметров,
    традиционное тесто, 1 пицца, 695 рублей.
    
    Доступны действия: добавить, убавить, удалить.
    
    Сырный соус, 0, элемент регулировки
    
    Барбекю, 1 штука, элемент регулировки
```

## Практика

1. Найдите в вашем приложении контролы, где уместен трейт `.adjustable`: степперы, слайдеры, любые места выбора значения из диапазона.
2. Выделите действия, которые можно вынести в `accessibilityCustomActions`: кнопки в ячейках, действия над карточками, множественные контролы у одного объекта.
3. Используйте `AXCustomContent` для ячеек с большим количеством данных: заказы, транзакции, профили.
