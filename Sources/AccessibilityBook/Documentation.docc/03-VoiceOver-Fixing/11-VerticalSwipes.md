# Вертикальные свайпы

Горизонтальные свайпы переключают фокус между элементами, а вертикальные — меняют значение внутри элемента.

## Трейт .adjustable

В UIKit есть несколько контролов, которые позволяют выбрать значение: `UIStepper`, `UISlider`, `UISegmentedControl`. Все они объединены одной идеей — вертикальный свайп меняет значение элемента.

За это отвечает трейт `.adjustable`. Когда вы добавляете его к элементу, происходит три вещи:

1. К описанию элемента добавляется фраза **«элемент регулировки»**.
2. VoiceOver подсказывает жест: «Смахните вверх или вниз одним пальцем, чтобы настроить значение».
3. При вертикальном свайпе вызываются методы `accessibilityIncrement()` или `accessibilityDecrement()`.

После свайпа VoiceOver не перечитывает всё описание элемента — **читается только новое значение**. Это делает переключение между значениями быстрым и ненавязчивым.

## UIStepper

Степпер по умолчанию состоит из двух кнопок «+» и «–». В VoiceOver это два отдельных элемента без подписи и без понимания текущего значения. Рядом со степпером обычно стоит лейбл с числом, но VoiceOver не знает, что они связаны.

Правильный подход — объединить степпер в один доступный элемент с трейтом `.adjustable`:

```swift
override public func awakeFromNib() {
    super.awakeFromNib()

    isAccessibilityElement = true
    accessibilityLabel = "Количество"
    accessibilityTraits = .adjustable
}

public override var accessibilityValue: String? {
    get { return "\(count) события" }
    set {}
}

override public func accessibilityIncrement() {
    count += 1
    count.limit(by: 0...10)
}

override public func accessibilityDecrement() {
    count -= 1
    count.limit(by: 0...10)
}
```

Теперь VoiceOver читает: «Количество, 3 события, элемент регулировки». Свайп вверх увеличивает значение, свайп вниз — уменьшает. После каждого свайпа читается только новое значение: «4 события», «5 событий».

Если значение не меняется после свайпа — например, достигнут конец диапазона — VoiceOver проигрывает характерный звуковой сигнал. Пользователь сразу понимает, что дальше крутить нельзя.

## UISlider

Слайдер работает по тому же принципу, что и степпер: вертикальный свайп меняет значение. Каждый свайп изменяет значение примерно на **10%** от общего диапазона.

Хорошая новость — `UISlider` уже имеет трейт `.adjustable` по умолчанию. Вам нужно только задать ему `accessibilityLabel`:

```swift
slider.accessibilityLabel = "Громкость"
```

VoiceOver прочитает: «Громкость, 50%, элемент регулировки».

Если шаг в 10% слишком грубый, есть альтернативный жест: **двойной тап с удержанием**. После двойного тапа не отпускайте палец — слайдер перейдёт в режим тонкой настройки. Двигайте палец вверх или вниз, чтобы плавно менять значение.

## UISegmentedControl

VoiceOver знает, что `UISegmentedControl` — это группа кнопок. При фокусе на сегменте он читает: «Тонкое, кнопка, 1 из 2». Видно, какой сегмент выбран и сколько их всего.

Но без контекста непонятно, что мы выбираем. Добавьте `accessibilityLabel` к самому контролу, чтобы задать контекст:

```swift
segmentedControl.accessibilityLabel = "Тесто"
```

Теперь при фокусе VoiceOver озвучит: «Тесто, тонкое, кнопка, 1 из 2». Стало понятнее.

Если сегментов немного — например, выбор размера пиццы — можно упростить контрол до одного элемента с трейтом `.adjustable`. Вместо нескольких кнопок пользователь получает один элемент и переключает значения вертикальными свайпами:

```swift
segmentedControl.isAccessibilityElement = true
segmentedControl.accessibilityLabel = "Размер"
segmentedControl.accessibilityTraits = .adjustable
segmentedControl.accessibilityValue = "Средняя"
```

Вертикальный свайп переключит значение на «Маленькая» или «Большая».

## Контекстные действия

Иногда у одного элемента несколько действий. Представьте ячейку корзины: в ней название товара, количество, кнопки «+», «–» и кнопка «Удалить». Без адаптации это пять отдельных элементов.

Лучше объединить ячейку в один элемент и добавить **контекстные действия** через `UIAccessibilityCustomAction`. Двойной тап активирует ячейку (действие по умолчанию), а вертикальный свайп переключает между дополнительными действиями:

- Активировать (по умолчанию, двойной тап)
- Добавить
- Убавить
- Удалить

```swift
let increase = UIAccessibilityCustomAction(
    name: "Добавить",
    actionHandler: { [unowned self] (action) -> Bool in
        // ... Увеличить количество
        return true
    })

let decrease = UIAccessibilityCustomAction(
    name: "Убавить",
    actionHandler: { [unowned self] (action) -> Bool in
        // ... Уменьшить количество
        return true
    })

let delete = UIAccessibilityCustomAction(
    name: "Удалить",
    actionHandler: { [unowned self] (action) -> Bool in
        // ... Удалить из корзины
        return true
    })

accessibilityCustomActions = [increase, decrease, delete]
```

Когда фокус стоит на ячейке, VoiceOver подсказывает: «Доступны действия». Вертикальный свайп переключает между действиями, двойной тап выполняет выбранное.

Стандартный жест **свайп-для-удаления** тоже превращается в контекстное действие и объединяется с вашими. Не нужно ничего делать дополнительно — VoiceOver подхватит его автоматически.

Этот же подход отлично работает для **постов в социальных сетях**. У поста может быть множество кнопок: лайк, комментарий, репост, сохранить, меню. Все эти кнопки становятся контекстными действиями одной ячейки. Пользователь не блуждает по десяткам кнопок, а быстро переключается между действиями вертикальным свайпом.

## AXCustomContent

Бывают ячейки, в которых очень много информации. Например, история заказов: название, дата, статус, сумма, адрес, способ оплаты. Если зачитать всё сразу — слишком долго. Если разбить на отдельные элементы — слишком много свайпов.

Для таких случаев есть `AXCustomContent`. Основная информация читается сразу при фокусе, а детали доступны через вертикальные свайпы.

Для начала импортируйте фреймворк:

```swift
import Accessibility
```

Затем реализуйте протокол `AXCustomContentProvider`:

```swift
class OrderCell: UITableViewCell, AXCustomContentProvider {

    var accessibilityCustomContent: [AXCustomContent]! {
        get {
            let status = AXCustomContent(label: "Статус", value: "Доставлен")
            status.importance = .high

            let date = AXCustomContent(label: "Дата", value: "12 марта")
            let address = AXCustomContent(label: "Адрес", value: "ул. Пушкина, 10")
            let payment = AXCustomContent(label: "Оплата", value: "Картой")

            return [status, date, address, payment]
        }
        set {}
    }
}
```

Свойство `.importance = .high` отмечает контент, который нужно зачитать сразу при фокусе вместе с основным описанием. Остальные элементы будут доступны через вертикальные свайпы в режиме «Ещё контент» ротора.

## Разные действия на одном экране

На одном экране могут сочетаться разные подходы. Возьмём экран корзины: для количества товара используем `customAction` (добавить, убавить, удалить), а для количества соуса — трейт `.adjustable` (просто больше или меньше).

Такой подход значительно сокращает количество элементов. Экран корзины, на котором было **31 элемент** для VoiceOver, после адаптации превращается в **6 элементов**. Навигация становится быстрой и понятной.

Выбирайте инструмент по задаче:

- **`.adjustable`** — когда нужно менять одно значение по шагам (количество, громкость, размер).
- **`customAction`** — когда у элемента несколько разных действий (добавить, удалить, поделиться).
- **`AXCustomContent`** — когда в элементе много информации и её нужно читать по частям.

## Практика

1. Найдите в вашем приложении контролы, где уместен трейт `.adjustable`: степперы, слайдеры, любые места выбора значения из диапазона.
2. Выделите действия, которые можно вынести в `accessibilityCustomActions`: кнопки в ячейках, действия над карточками, множественные контролы у одного объекта.
3. Используйте `AXCustomContent` для ячеек с большим количеством данных: заказы, транзакции, профили.
4. Проверьте результат в VoiceOver: посчитайте количество свайпов до и после адаптации.
