# Подписи

Как правильно подписывать элементы интерфейса, чтобы VoiceOver читал их понятно и корректно.

## Подписи 

![Справочник жестов VoiceOver.](labeling-text)

Обычные подписи уже адаптированы для VoiceOver: их размер задаст фрейм для фокуса, а текст в лейбле — это текст самой надписи.

> Tip: Текстовая надпись – это базовый элемент доступности.

Чаще всего текст для незрячего не отличается от того, что уже есть в интерфейсе, но вы можете задавать другой текст.

```swift
accessibilityLabel = "Пицца «Пепперони»"
```

> Tip: для заголовков еще надо добавлять трейт .header.

## Раскрыть сокращения

VoiceOver читает текст из `accessibilityLabel` ровно так, как он написан. Это значит, что сокращения и символы будут прочитаны *буквально*. 

![Пицца с названием](labeling-text-advanced)

Например, надпись «30 см» будет прочитана как «тридцать сэмэ», а не «тридцать сантиметров». Цена «30 ₽» превратится в «тридцать знак рубля» вместо «тридцать рублей». VoiceOver не умеет раскрывать сокращения и подставлять правильные названия валют — он читает то, что видит.

Чтобы исправить это, задайте `accessibilityLabel` с полным текстом:

```swift
priceLabel.text = "30 ₽"
priceLabel.accessibilityLabel = "30 рублей"
```

Если нужно объединить несколько строк в одну подпись, используйте запятую — VoiceOver сделает небольшую паузу и правильно расставит интонацию:

```swift
cell.accessibilityLabel = "Пепперони, 25 см, тонкое тесто, 625 рублей"
```

Для удобства можно завести структуру, которая хранит видимый и доступный текст рядом:

```swift
struct AccessibleText {
    let visibleText: String
    let accessibleText: String
}

let size = AccessibleText(
    visibleText: "30 см",
    accessibleText: "30 сантиметров"
)

let price = AccessibleText(
    visibleText: "30 ₽",
    accessibleText: "30 рублей"
)
```

Так бэкенд или дизайн-система всегда могут предоставить правильный текст для VoiceOver вместе с отображаемым. Подобное пригодится и для backend-driven-ui.

### Множественное число

В русском языке число влияет на форму слова: 1 рубль, 2 рубля, 5 рублей. Если вы подставляете число в строку, нужно учитывать склонение.

Используйте `Localizable.stringsdict` для правильного склонения:

![Пример файла локализации](localizable)

```swift
String(format: NSLocalizedString("%d рублей", comment: ""), price)
```

Теперь «1» превратится в «1 рубль», «2» — в «2 рубля», а «5» — в «5 рублей».

Кроме того, `NSStringVariableWidthRuleType` помогает выбирать разные тексты в зависимости от доступной ширины. Это полезно, когда в интерфейсе нужна короткая форма, а для VoiceOver — полная:

![Пример описания для разной ширины](width-rule)

### Когда подписи ломаются

Если вы рисуете текст через `CATextLayer`, то информация о доступности теряется. Восстановить её можно самостоятельно, указав параметры. Разберем такой случай и посмотрим, что UILabel делает за нас.

Для начала, отметим, что элемент доступен, а значит, на него можно поставить фокус. 

```swift
isAccessibilityElement = true
```

> Note: Если значение будет false, то VoiceOver попытается найти доступный элемент среди дочерних.

Затем нужно подписать элемент, для описания возьмем текст в элементе. Этот текст VoiceOver прочитает, когда фокус попадет на контрол.

```swift 
accessibilityLabel = text
```

В конце нужно задать фрейм элемента: так фокус будет виден на экране и можно будет навести касанием. Фокус нужен не только для незрячих и VoiceOver, но и для Switch Control.

```swift
accessibilityFrame = frameInScreenCoordinates
```

Готово, доступность в самом простом виде заработала.

@Comment {
    Добавить ссылку на заголовки
}

## Как называть элементы?

Хорошая подпись обладает несколькими свойствами:

1. **Краткое описание.** Одно слово или короткая фраза: «Добавить», «Корзина», «Настройки». Незрячие слушают VoiceOver на высокой скорости — длинные описания замедляют работу.

2. **Без указания типа.** Не пишите «кнопка Закрыть» или «картинка Пепперони» — тип элемента добавляется автоматически через трейты. Если написать «Кнопка Закрыть», VoiceOver прочитает «Кнопка Закрыть, кнопка».

3. **Начинается с заглавной буквы.** VoiceOver может использовать регистр для интонации.

4. **Без точки в конце.** Точка создаёт ненужную паузу. Если нужно перечислить несколько свойств, используйте запятую.

5. **На языке пользователя.** Если интерфейс на русском — подписи на русском. «NEW» в `accessibilityLabel` прозвучит как «нью», лучше написать «Новинка».

6. **Составные подписи через запятую.** VoiceOver делает короткую паузу на запятой — используйте это для структурирования:

```swift
cell.accessibilityLabel = "Пепперони, 25 см, тонкое тесто"
```

> Tip: Хорошая проверка: представьте, что вы описываете элемент по телефону другу. Как бы вы назвали его одним-двумя словами?
