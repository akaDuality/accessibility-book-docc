# Подписи

Как правильно подписывать элементы интерфейса, чтобы VoiceOver читал их понятно и корректно.

## Подписи 

![Справочник жестов VoiceOver.](labeling-text)

Обычные подписи уже адаптированы для VoiceOver: их размер задаст фрейм для фокуса, а текст в лейбле — это текст самой надписи.

> Tip: Текстовая надпись – это базовый элемент доступности.

Чаще всего текст для незрячего не отличается от того, что уже есть в интерфейсе, но вы можете задавать другой текст.

@TabNavigator {
    @Tab("UIKit") {
        ```swift
        label.accessibilityLabel = "Пицца «Пепперони»"
        ```
    }
    @Tab("SwiftUI") {
        ```swift
        Text("Пепперони")
            .accessibilityLabel("Пицца «Пепперони»")
        ```
    }
}

> Tip: для заголовков добавляйте трейт `.header` — пользователи переключаются между заголовками через ротор, как зрячие листают экран глазами.

Применяйте трейт для заголовков секций в списках, заголовка экрана и крупных текстовых блоков:

@TabNavigator {
    @Tab("UIKit") {
        ```swift
        titleLabel.accessibilityTraits = .header
        ```
    }
    @Tab("SwiftUI") {
        ```swift
        Text("Меню")
            .accessibilityAddTraits(.isHeader)
        ```
    }
}

## Раскрыть сокращения

VoiceOver читает текст из `accessibilityLabel` ровно так, как он написан. Это значит, что сокращения и символы будут прочитаны *буквально*. 

![Пицца с названием](labeling-text-advanced)

Например, надпись «30 см» будет прочитана как «тридцать сэмэ», а не «тридцать сантиметров». Цена «30 ₽» превратится в «тридцать знак рубля» вместо «тридцать рублей». VoiceOver не умеет раскрывать сокращения и подставлять правильные названия валют — он читает то, что видит.

Чтобы исправить это, задайте `accessibilityLabel` с полным текстом:

@TabNavigator {
    @Tab("UIKit") {
        ```swift
        priceLabel.text = "30 ₽"
        priceLabel.accessibilityLabel = "30 рублей"
        ```
    }
    @Tab("SwiftUI") {
        ```swift
        Text("30 ₽")
            .accessibilityLabel("30 рублей")
        ```
    }
}

Если нужно объединить несколько строк в одну подпись, используйте запятую — VoiceOver сделает небольшую паузу и правильно расставит интонацию:

@TabNavigator {
    @Tab("UIKit") {
        ```swift
        cell.accessibilityLabel = "Пепперони, 25 сантиметров, тонкое тесто, 625 рублей"
        ```
    }
    @Tab("SwiftUI") {
        ```swift
        PizzaCell(pizza: pepperoni)
            .accessibilityLabel("Пепперони, 25 сантиметров, тонкое тесто, 625 рублей")
        ```
    }
}

Для удобства можно завести структуру, которая хранит видимый и доступный текст рядом:

```swift
struct AccessibleText {
    let visibleText: String
    let accessibleText: String
}

let size = AccessibleText(
    visibleText: "30 см",
    accessibleText: "30 сантиметров"
)

let price = AccessibleText(
    visibleText: "30 ₽",
    accessibleText: "30 рублей"
)
```

Так бэкенд или дизайн-система всегда могут предоставить правильный текст для VoiceOver вместе с отображаемым. Подобное пригодится и для backend-driven-ui.

### Числа: телефоны, карты, коды

VoiceOver читает длинные числа целиком — это часто звучит непонятно для номеров телефонов, карт и кодов подтверждения. Разбивайте такие числа пробелами, чтобы VoiceOver сделал паузы и прочитал по группам цифр.

@TabNavigator {
    @Tab("UIKit") {
        ```swift
        // Без обработки длинное число читается одним блоком
        phoneLabel.text = "+79991234567"

        // С пробелами VoiceOver прочитает по группам
        phoneLabel.accessibilityLabel = "+7 999 123 45 67"
        ```
    }
    @Tab("SwiftUI") {
        ```swift
        Text("+79991234567")
            .accessibilityLabel("+7 999 123 45 67")
        ```
    }
}

Для номеров карт удобно разбивать по 4 цифры, для OTP — по одной:

@TabNavigator {
    @Tab("UIKit") {
        ```swift
        cardLabel.accessibilityLabel = "4242 4242 4242 4242"
        otpLabel.accessibilityLabel = "1 2 3 4"
        ```
    }
    @Tab("SwiftUI") {
        ```swift
        Text("4242424242424242")
            .accessibilityLabel("4242 4242 4242 4242")

        Text("1234")
            .accessibilityLabel("1 2 3 4")
        ```
    }
}

### Множественное число

В русском языке число влияет на форму слова: 1 рубль, 2 рубля, 5 рублей. Если вы подставляете число в строку, нужно учитывать склонение.

Используйте `Localizable.stringsdict` для правильного склонения:

![Пример файла локализации](localizable)

```swift
String(format: NSLocalizedString("%d рублей", comment: ""), price)
```

Теперь «1» превратится в «1 рубль», «2» — в «2 рубля», а «5» — в «5 рублей».

Кроме того, `NSStringVariableWidthRuleType` помогает выбирать разные тексты в зависимости от доступной ширины. Это полезно, когда в интерфейсе нужна короткая форма, а для VoiceOver — полная:

![Пример описания для разной ширины](width-rule)

### Пунктуация и паузы

Знаки препинания в `accessibilityLabel` влияют на интонацию и темп VoiceOver:

- **Запятая** — короткая пауза, следующая часть читается чуть другим тоном
- **Двоеточие, точка** — длинная пауза
- **Тире** — короткая пауза, само не озвучивается
- **Скобки и кавычки** — обычно читаются буквально: «открывающая скобка», «кавычка»

Сравните, как звучит одна и та же информация при разной пунктуации:

@TabNavigator {
    @Tab("UIKit") {
        ```swift
        // Точки дробят на отдельные предложения, чтение замедляется
        cell.accessibilityLabel = "Пепперони. 25 см. Тонкое тесто. 625 рублей."

        // Запятые дают плавную интонацию с короткими паузами
        cell.accessibilityLabel = "Пепперони, 25 см, тонкое тесто, 625 рублей"

        // Скобки прочитаются буквально: «Пепперони, открывающая скобка, острая, закрывающая скобка»
        cell.accessibilityLabel = "Пепперони (острая)"

        // Лучше использовать запятую
        cell.accessibilityLabel = "Пепперони, острая"
        ```
    }
    @Tab("SwiftUI") {
        ```swift
        // Точки дробят на отдельные предложения, чтение замедляется
        cellView.accessibilityLabel("Пепперони. 25 см. Тонкое тесто. 625 рублей.")

        // Запятые дают плавную интонацию с короткими паузами
        cellView.accessibilityLabel("Пепперони, 25 см, тонкое тесто, 625 рублей")

        // Скобки прочитаются буквально
        cellView.accessibilityLabel("Пепперони (острая)")

        // Лучше использовать запятую
        cellView.accessibilityLabel("Пепперони, острая")
        ```
    }
}

Используйте запятую для структурирования и не злоупотребляйте точками — они дробят описание на отдельные предложения и замедляют чтение.

### Когда подписи ломаются

Если вы рисуете текст через `CATextLayer`, то информация о доступности теряется. Восстановить её можно самостоятельно, указав параметры. Разберем такой случай и посмотрим, что UILabel делает за нас.

Для начала, отметим, что элемент доступен, а значит, на него можно поставить фокус. 

```swift
isAccessibilityElement = true
```

> Note: Если значение будет false, то VoiceOver попытается найти доступный элемент среди дочерних.

Затем нужно подписать элемент, для описания возьмем текст в элементе. Этот текст VoiceOver прочитает, когда фокус попадет на контрол.

```swift 
accessibilityLabel = text
```

В конце нужно задать фрейм элемента: так фокус будет виден на экране и можно будет навести касанием. Фокус нужен не только для незрячих и VoiceOver, но и для Switch Control.

```swift
accessibilityFrame = frameInScreenCoordinates
```

Готово, доступность в самом простом виде заработала.

## Произношение и язык

VoiceOver выбирает голос по языку системы. Если в русской подписи появляется английское слово, оно прочитается с русским произношением: «iPhone» как «ипхон», «AirPods» как «аирподс», «Wi-Fi» как «уи-фи». Для брендовых названий это самый частый источник кривого озвучивания.

Если вся подпись на другом языке, укажите его через `accessibilityLanguage`:

@TabNavigator {
    @Tab("UIKit") {
        ```swift
        button.accessibilityLabel = "Settings"
        button.accessibilityLanguage = "en-US"
        ```
    }
    @Tab("SwiftUI") {
        ```swift
        var label = AttributedString("Settings")
        label.accessibilitySpeechLanguage = "en-US"

        Button("Settings") { }
            .accessibilityLabel(label)
        ```
    }
}

Если язык меняется внутри строки, используйте `accessibilityAttributedLabel` с атрибутом `.accessibilitySpeechLanguage`:

@TabNavigator {
    @Tab("UIKit") {
        ```swift
        let label = NSMutableAttributedString(string: "Подключите iPhone")
        label.addAttribute(
            .accessibilitySpeechLanguage,
            value: "en-US",
            range: NSRange(location: 11, length: 6)
        )
        button.accessibilityAttributedLabel = label
        ```
    }
    @Tab("SwiftUI") {
        ```swift
        var label = AttributedString("Подключите ")
        var brand = AttributedString("iPhone")
        brand.accessibilitySpeechLanguage = "en-US"
        label.append(brand)

        Button("Подключите iPhone") { }
            .accessibilityLabel(label)
        ```
    }
}

Теперь VoiceOver переключит голос на английский на слове «iPhone» и прочитает его правильно.

> Tip: Проверяйте, как звучат брендовые названия и иностранные слова — на слух ошибку легко пропустить.

## Как проверить подпись

После изменения подписи нужно убедиться, что VoiceOver читает её именно так, как вы хотели. 

### RocketSim 

Удобнее всего было бы проверять прямо в симуляторе и видеть работу VoiceOver на каждом запуске. Для этого можно использовать [RocketSim](https://www.rocketsim.app) — он показывает все доступные элементы, их подписи, трейты и порядок элементов рядом с симулятором.  

Приложение позволяет упростить работу и [с другимим настройками доступности](https://www.rocketsim.app/features/accessibility/)

### Панель субтитров

Если проверяете на телефоне, то удобно включить **субтитры** (см. <doc:02-EnablingVoiceOver>).

Субтитры показывают текст, который VoiceOver произносит прямо сейчас. Не нужно ничего слушать — наведите фокус на элемент и прочитайте строку внизу экрана. Это особенно удобно при отладке: меняете подпись, перезапускаете приложение, проверяете строку — и сразу видите результат.

> Tip: Включайте субтитры, когда работаете с длинными подписями, числами или иностранными словами — на слух легко пропустить ошибку.

## Как называть элементы?

Хорошая подпись обладает несколькими свойствами:

1. **Краткое описание.** Одно слово или короткая фраза: «Добавить», «Корзина», «Настройки». Незрячие слушают VoiceOver на высокой скорости — длинные описания замедляют работу.

2. **Без указания типа.** Не пишите «кнопка Закрыть» или «картинка Пепперони» — тип элемента добавляется автоматически через трейты. Если написать «Кнопка Закрыть», VoiceOver прочитает «Кнопка Закрыть, кнопка».

3. **Начинается с заглавной буквы.** VoiceOver может использовать регистр для интонации.

4. **Без точки в конце.** Точка создаёт ненужную паузу. Если нужно перечислить несколько свойств, используйте запятую.

5. **На языке пользователя.** Если интерфейс на русском — подписи на русском. «NEW» в `accessibilityLabel` прозвучит как «нью», лучше написать «Новинка».

6. **Составные подписи через запятую.** VoiceOver делает короткую паузу на запятой — используйте это для структурирования:

```swift
cell.accessibilityLabel = "Пепперони, 25 см, тонкое тесто"
```

7. **Не оставляйте подпись пустой.** Если `accessibilityLabel = nil` или пустая строка, VoiceOver прочитает только трейт («кнопка», «изображение») без имени. Пользователь не поймёт, что это за элемент. Особенно опасно для иконок без текста.

8. **Близко к видимому тексту.** Подпись должна быть похожа на то, что написано на экране. «См. все» → «Смотреть все» — нормально. «Подробнее» → «Перейти на экран характеристик товара» — плохо: зрячий помощник, читая подпись вслух, не поймёт, что показано на экране.

> Tip: Хорошая проверка: представьте, что вы описываете элемент по телефону другу. Как бы вы назвали его одним-двумя словами?
