# Набор текста

Как VoiceOver работает с полями ввода, экранной клавиатурой и клавиатурой Брайля.

## Поля ввода

Когда VoiceOver фокусируется на текстовом поле, он читает несколько вещей по порядку:

1. **Label** — название поля: «Имя», «Электронная почта», «Пароль».
2. **Value** — текущий текст в поле. Если пустое — ничего не читается.
3. **Тип** — «текстовое поле» или «защищённое текстовое поле».
4. **Hint** — подсказка: «Введите ваш email».

```swift
let emailField = UITextField()
emailField.accessibilityLabel = "Электронная почта"
emailField.accessibilityHint = "Введите адрес для регистрации"
emailField.placeholder = "example@mail.ru"
```

Для поля ввода `placeholder` автоматически используется как `accessibilityLabel`, если вы не задали его явно. Но полагаться на это не стоит — плейсхолдер часто содержит пример значения, а не название поля. Лучше задавать `accessibilityLabel` отдельно.

### Ввод текста

Двойной тап на текстовом поле открывает клавиатуру и начинает редактирование. VoiceOver скажет «выполняется редактирование» и переместит фокус на клавиатуру.

### Диктовка

VoiceOver поддерживает жест Magic Tap — **двойной тап двумя пальцами**. В контексте текстового поля этот жест запускает диктовку: пользователь говорит текст, а система его вводит. Это самый быстрый способ набора для незрячих.

### Не подменяйте значение

Если поле форматирует текст при вводе (например, номер телефона или банковская карта), не подставляйте форматированный текст обратно в поле через код. VoiceOver прочитает каждое изменение, и пользователь услышит лишний текст при каждом нажатии клавиши.

```swift
// Плохо: VoiceOver прочитает изменение при каждом символе
func textField(_ textField: UITextField,
               shouldChangeCharactersIn range: NSRange,
               replacementString string: String) -> Bool {
    let formatted = format(phone: newText)
    textField.text = formatted // VoiceOver зачитает весь текст
    return false
}

// Лучше: форматируйте отображение отдельно от значения
// или используйте accessibilityValue для озвучивания
```

Если форматирование необходимо, управляйте `accessibilityValue` вручную, чтобы VoiceOver читал только осмысленный результат.

## Экранная клавиатура

У VoiceOver есть три режима набора текста на экранной клавиатуре. Переключаются они через ротор (жест вращения двумя пальцами).

### Стандартный набор

Режим по умолчанию. Пользователь свайпает по клавиатуре, VoiceOver читает букву под пальцем. Когда нужная буква найдена, двойной тап в любом месте экрана вводит её. Надёжно, но медленно.

### Набор вслепую

Пользователь касается клавиатуры и слушает, какая буква под пальцем. Когда нужная буква найдена, достаточно **поднять палец** — буква введётся сразу. Быстрее стандартного режима, потому что не нужен второй тап.

### Прямой набор

Клавиатура работает так, как будто VoiceOver выключен: нажали на букву — она введена. Этот режим используют те, кто хорошо помнит расположение клавиш. Самый быстрый, но требует навыка.

### Трейт .keyboardKey

Если вы создаёте кастомные кнопки рядом с клавиатурой (например, панель автодополнения или кастомную вью над клавиатурой), добавьте трейт `.keyboardKey`:

```swift
suggestionButton.accessibilityTraits = .keyboardKey
```

Этот трейт говорит VoiceOver, что элемент ведёт себя как клавиша: в режиме набора вслепую он будет активироваться по поднятию пальца, а не по двойному тапу.

## Клавиатура Брайля

VoiceOver включает специальный режим — **клавиатура Брайля на экране**. Активируется через ротор.

### Как она устроена

Телефон нужно держать экраном от себя — как будто показываете экран собеседнику. На экране появляются 6 точек — по 3 с каждого края, как на брайлевской ячейке. Каждой точке соответствует палец.

Пользователь нажимает нужные точки одновременно, и система вводит букву. Это быстрый и точный способ набора для людей, знакомых с азбукой Брайля.

### Жесты

У клавиатуры Брайля есть набор жестов для управления:

| Жест | Действие |
|------|----------|
| Свайп вправо одним пальцем | Пробел |
| Свайп влево одним пальцем | Удалить символ |
| Свайп вправо двумя пальцами | Ввод (Enter) |
| Свайп влево двумя пальцами | Удалить слово |
| Свайп вверх двумя пальцами | Переключить язык |
| Свайп вниз тремя пальцами | Закрыть клавиатуру Брайля |

### Особенности

Клавиатура Брайля особенно удобна в шумном окружении, когда голосовой ввод невозможен. Она работает бесшумно и не требует внешнего оборудования.

> Tip: Если ваше приложение использует кастомную клавиатуру (Custom Keyboard Extension), убедитесь, что все клавиши доступны для VoiceOver и имеют правильные подписи. Стандартная клавиатура iOS уже полностью адаптирована.
