# Voice Control

Голосовое управление iOS-приложениями для людей с моторными нарушениями.

## Обзор

Voice Control появился в iOS 13 и позволяет людям с моторными нарушениями управлять приложениями с помощью голоса. Функция накладывает на интерфейс надписи, цифры или сетку, помогая пользователю указать нужный элемент.

В отличие от VoiceOver, Voice Control менее требователен к адаптации: графический интерфейс остаётся на экране, пользователь видит все элементы и взаимодействует с ними голосом. Телефон использует камеру, чтобы убедиться, что вы смотрите на экран, когда произносите команду.

На данный момент Voice Control работает только на английском языке.

## Виды навигации

Voice Control предлагает три режима для определения элемента, с которым нужно взаимодействовать.

### Показать надписи

На каждом элементе интерфейса появляется его название. Пользователь произносит название элемента, чтобы нажать на него. Этот режим лучше всего работает с адаптированными приложениями, где у каждого элемента есть осмысленная подпись.

### Показать цифры

Все интерактивные элементы нумеруются. Чтобы нажать на элемент, достаточно сказать, например, "tap 6". Этот режим удобен, когда у элементов длинные или сложные названия.

### Показать сетку

На экран накладывается сетка. Пользователь называет номер столбца, затем номер строки, чтобы уточнить область. Сетка делится на более мелкие части, пока не будет выбран нужный элемент. Этот режим хорошо подходит для работы с картами и другими элементами без чёткой структуры.

Voice Control также косвенно проверяет качество VoiceOver-адаптации: в режиме надписей сразу видно, у каких элементов отсутствуют подписи.

## Снова про adjustable

Voice Control иногда конфликтует с оптимизациями, сделанными для VoiceOver. Например, `UISegmentedControl`, обёрнутый как `.adjustable`, хуже работает с Voice Control: пользователю проще просто нажать на нужную кнопку голосом, чем использовать жесты свайпа.

Решение — применять `.adjustable` только тогда, когда активен VoiceOver:

```swift
override public var accessibilityTraits: UIAccessibilityTraits {
    get {
        UIAccessibility.isVoiceOverRunning ? .adjustable : .none
    }
    set {}
}

public override var isAccessibilityElement: Bool {
    get {
        UIAccessibility.isVoiceOverRunning
    }
    set {}
}
```

Такой же подход упрощает UI-тесты: вместо свайпов по adjustable-элементу проще нажимать на отдельные кнопки напрямую.

## Альтернативные названия

Голосовые команды не всегда точно совпадают с текстом на кнопке. Пользователь может сказать "купить", "в корзину" или "добавить", имея в виду одну и ту же кнопку. Для этого существует свойство `accessibilityUserInputLabels`:

```swift
putToCartButton.accessibilityUserInputLabels = ["Купить", "Добавить", "В корзину"]
```

Это особенно полезно для ячеек меню, где VoiceOver-метка включает цену и описание. Для Voice Control достаточно указать только название продукта в `accessibilityUserInputLabels`, чтобы пользователю не приходилось произносить всю строку целиком.

## Голосовые команды

Voice Control поддерживает множество встроенных команд: прокрутка, переключение между приложениями, управление громкостью и другие. Полный список команд доступен на сайте [imore.com](https://imore.com).

Помимо встроенных, можно создавать собственные команды — например, для ввода адреса или выполнения повторяющихся действий. Apple выпустила видео о создании пользовательских голосовых команд.
