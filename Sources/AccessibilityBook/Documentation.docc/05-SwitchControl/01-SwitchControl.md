# Switch Control

Самая гибкая технология доступности iOS для людей с тяжёлыми моторными нарушениями.

## Обзор

VoiceOver помогает незрячим пользователям. Voice Control помогает тем, кто не может касаться экрана, но может говорить. Switch Control предназначен для людей, которые не могут ни касаться экрана, ни отчётливо говорить. Это самый экстремальный, но и самый гибкий инструмент доступности в iOS.

Принцип работы прост: любое устройство ввода — кнопка, датчик, сенсор — подключается к iPhone и отправляет сигнал, который привязывается к команде. Кнопки могут быть физическими: пользователь нажимает их кулаком, пяткой или затылком. Датчики крепятся к мышцам и реагируют на сокращение. Камера iPhone может отслеживать движение бровей. Любое воспроизводимое движение становится способом управления телефоном.

## Как управлять

Начнём с самого сложного случая: у пользователя есть только один сигнал. К iPhone подключена одна кнопка или один датчик.

С одной кнопкой используется автосканирование: Switch Control автоматически перемещает фокус по элементам интерфейса, а пользователь нажимает кнопку, чтобы выбрать текущий элемент. Это медленно, но работает.

С двумя кнопками становится проще: одна перемещает фокус вперёд, вторая выбирает элемент. С тремя — вперёд, назад и выбор. Каждая дополнительная кнопка значительно улучшает удобство использования: можно назначить прокрутку, вызов меню, возврат на предыдущий экран и другие действия.

## Настройка

Switch Control настраивается в Настройки -> Универсальный доступ -> Switch Control -> Переключатели.

Источники сигнала:

- **Внешнее устройство.** Клавиатура, специализированные кнопки или любое Bluetooth-устройство, которое умеет отправлять нажатия.
- **Касание экрана.** Экран iPhone работает как одна большая кнопка — любое касание отправляет сигнал.
- **Камера.** Фронтальная камера отслеживает поворот головы влево или вправо и преобразует его в сигнал.
- **Касание задней панели.** Двойное или тройное касание задней панели iPhone (Back Tap).

После добавления источника нужно назначить ему действие. Для тестирования на симуляторе или с внешней клавиатурой удобно использовать стрелки для навигации и пробел для выбора: стрелка вправо — переход к следующему элементу, стрелка влево — к предыдущему, пробел — выбор, удержание пробела — вызов контекстного меню.

Минимальная конфигурация — один переключатель с действием «Выбрать». Switch Control включит автосканирование и будет автоматически перемещать фокус, а пользователь нажатием подтвердит выбор.

## Навигация

Switch Control поддерживает два режима фокуса: **режим объектов** и **скользящий курсор**.

### Режим объектов

Режим объектов — основной способ навигации. Switch Control использует дерево доступности (то же самое, что и VoiceOver), поэтому адаптированные приложения работают значительно лучше.

На домашнем экране (Springboard) сканирование работает по уровням: сначала Switch Control выделяет строки иконок, пользователь выбирает нужную строку, затем сканирование переключается на отдельные иконки внутри этой строки. Это значительно сокращает количество нажатий по сравнению с перебором всех иконок подряд.

### Скользящий курсор

Скользящий курсор напоминает игру с автоматом-краном. Горизонтальная полоса начинает двигаться сверху вниз по экрану. Пользователь нажимает кнопку, чтобы зафиксировать вертикальную позицию. Затем внутри этой полосы начинает двигаться вертикальная линия слева направо — пользователь нажимает ещё раз, чтобы зафиксировать горизонтальную позицию. В результате получается точная точка на экране. Этот режим хорошо подходит для неадаптированных приложений, где дерево доступности настроено плохо.

После выбора элемента — в любом из режимов — появляется контекстное меню с доступными действиями. В этом меню отображаются и элементы `UIAccessibilityCustomAction`, если они определены для выбранного элемента. Для пользовательских действий можно задавать иконки.

## Режим объектов

Switch Control повышает эффективность навигации за счёт группировки элементов. Рассмотрим пример с клавиатурой.

На стандартной клавиатуре 23 клавиши. Без группировки пользователю пришлось бы ждать, пока фокус дойдёт до нужной буквы — в худшем случае 23 нажатия. Но Switch Control группирует клавиши: сначала три строки (3 группы), затем внутри строки — подгруппы по 3 клавиши. Чтобы выбрать букву «ф», потребуется всего 7 действий вместо 23.

Группировкой можно управлять через свойство `accessibilityNavigationStyle`:

```swift
// Элемент сам решает, как группировать дочерние элементы
container.accessibilityNavigationStyle = .automatic

// Каждый дочерний элемент сканируется отдельно
container.accessibilityNavigationStyle = .separate

// Все дочерние элементы объединены в одну группу
container.accessibilityNavigationStyle = .combined
```

По умолчанию используется `.automatic` — система сама решает, нужна ли группировка. Значение `.combined` объединяет все дочерние элементы в один: фокус выделяет весь контейнер целиком, а после выбора — переходит к сканированию элементов внутри. Значение `.separate` отключает группировку: каждый элемент сканируется независимо.

Группировку особенно полезно настраивать для сложных экранов с большим количеством элементов. Правильная иерархия групп сокращает количество нажатий в разы.

## Скользящий курсор

Скользящий курсор используется, когда режим объектов не может добраться до нужного элемента — например, в неадаптированном приложении или в приложении со сложной кастомной отрисовкой.

Процесс работает в четыре шага:

1. Горизонтальная полоса начинает сканировать экран сверху вниз. Пользователь нажимает кнопку, фиксируя вертикальную область.
2. Внутри зафиксированной полосы тонкая горизонтальная линия уточняет позицию. Пользователь нажимает ещё раз.
3. Вертикальная полоса повторяет процесс для горизонтальной оси — сначала широкая область, затем точная позиция.
4. После определения точки появляется контекстное меню, которое уже работает в режиме объектов.

Механика напоминает автомат-кран: сначала грубая наводка, затем точная. Два этапа по каждой оси позволяют попасть в любую точку экрана с высокой точностью.

## Контекстные действия

Switch Control понимает все `UIAccessibilityCustomAction`, которые используются для VoiceOver. Это значит, что адаптация для VoiceOver автоматически улучшает и Switch Control.

Контекстное меню показывает системные действия (нажатие, прокрутка, возврат) и пользовательские действия. Для пользовательских действий можно задавать иконки:

```swift
let archiveAction = UIAccessibilityCustomAction(
    name: "Архивировать",
    image: UIImage(systemName: "archivebox")
) { _ in
    self.archiveItem()
    return true
}

let deleteAction = UIAccessibilityCustomAction(
    name: "Удалить",
    image: UIImage(systemName: "trash")
) { _ in
    self.deleteItem()
    return true
}

cell.accessibilityCustomActions = [archiveAction, deleteAction]
```

Если у действия нет иконки, в контекстном меню отображается просто точка. Иконки делают меню нагляднее и помогают пользователю быстрее находить нужное действие. Поскольку пользователи Switch Control видят экран, визуальная составляющая меню для них важна.
